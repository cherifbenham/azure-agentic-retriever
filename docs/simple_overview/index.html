What This Project Does

The repository contains a complete sample application that lets people chat with their own documents, powered by Azure OpenAI and Azure AI Search. Think of it as a customizable assistant: you upload company files, it builds a smart search index, and then it answers questions with citations.

The app includes a polished React frontend, a Python backend, and ready-to-use Azure infrastructure scripts. Sample content from a fictional company (Contoso Electronics) is bundled so you can explore the experience instantly.

Key Features in Plain Words

Chat or Ask

Use a chat window for natural conversations or a single-question page when you only need quick answers.

Shows Its Work

Every response comes with links to the documents it used, plus a trace of the reasoning steps.

Easy Experimenting

Built-in settings let you change model behavior, test ideas, and compare results without coding.

Azure Ready

Scripts deploy everything to Azure safely, from OpenAI models to data search and monitoring tools.

How It Works Behind the Scenes

  1. Ingest documents. You run a prep script that reads PDFs, Word files, web pages, and more, then stores clean snippets in Azure AI Search.
  2. Ask a question. The backend rewrites the question if needed, pulls the most relevant snippets, and sends them to Azure OpenAI.
  3. Get an answer with sources. The model writes a response that cites the documents it used, so you can double-check the facts.

What You Need to Try It

  • An Azure account with permission to deploy resources, plus enough credits to run OpenAI and search services.
  • Developer tools on your computer: Python 3.9+, Node.js 20+, Git, and the Azure Developer CLI (`azd`).
  • The sample data included in the repository, or your own files if you want to test with real content.

This sample is designed for learning. Before using it in production, follow the security checklist in docs/productionizing.md and review the Azure OpenAI landing zone guidance.

Quick Start Guide

Here is the simplest way to see the app working with the included sample data:

  1. Prepare your tools. Install the Azure Developer CLI, Python 3.9+, Node.js 20+, and Git.
  2. Get the code. In an empty folder run: 
    azd init -t azure-search-openai-demo
    This command downloads the repo and sets up a new environment.
  3. Load the example documents. From the project root run: 
    scripts/prepdocs.sh
    (On Windows use scripts/prepdocs.ps1.) This script sends the Contoso sample files to Azure AI Search.
  4. Launch everything. Sign in and deploy with: 
    azd auth login
azd env new
azd up
    When the command finishes it prints the public URL. Open it to chat with the data.

Prefer to run locally? Start the backend with azd run watch --service backend and the frontend with npm run dev inside app/frontend. Both commands hot-reload as you make changes.

Finished experimenting? Run azd down or delete the resource group in Azure to avoid extra charges.

Other Fast Start Options

GitHub Codespaces

Launch a cloud dev environment from your browser. Click the Codespaces badge in the main README, wait for setup, then open a terminal and follow the same azd commands above.

VS Code Dev Containers

Start Docker Desktop, open the repo in VS Code using the Dev Containers extension, and the container installs all tools for you. Use the integrated terminal to run scripts/prepdocs.sh and azd up.

Customize Locally

After azd env new, set optional environment tweaks (like auth or vision support) with azd env set. Update the files in app, then redeploy with azd deploy.

How We Measure Quality

The evals/results folder documents baseline tests run with current model settings. For example, the default setup (June 2025) uses the GPT-4.1-mini chat model with a powerful embedding model. Evaluations are run with GPT-4o to score answers, so you can compare future experiments against a known baseline.
